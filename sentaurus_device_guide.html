Sentaurus Device User Guide

Reference guide for Sentaurus Device (Version T-2022.03). This document covers starting the tool, defining physics, solving equations, mixed-mode simulation, and key numeric parameters.

Getting Started

Overview & Functionality

Sentaurus Device is an advanced 1D, 2D, and 3D device simulator capable of modeling the electrical, thermal, and optical characteristics of semiconductor devices. It supports single/multiple devices, mixed-mode circuit simulation, nonvolatile memory, hydrodynamic transport, and thermodynamic models.

Starting Sentaurus Device

To run a simulation from the command line, provide the command filename:

sdevice my_device.cmd
sdevice -P my_device.cmd  # Preprocess macro parsing
sdevice -ver              # Print version
sdevice -h                # Print help

Specifying Physical Devices

Reading a Structure

The mesh and doping profile are typically generated by Sentaurus Structure Editor or Sentaurus Process and loaded via the File section:

File {
  Grid      = "mosfet_msh.tdr"
  Parameter = "models.par"
}

You can adjust dimensions using AreaFactor or specify a CoordinateSystem (e.g., Cylindrical).

Doping & Materials

Regions are defined in the .tdr file. Materials are evaluated hierarchically. If a material is not built-in, you can specify user-defined mole fraction materials (e.g., Molefraction.txt) or map them via datexcodes.txt.

Physics Model Hierarchy

Sentaurus Device applies physical models in a strict hierarchy:

  1. Global (applies to all regions)
  2. Material-specific
  3. Region-specific
  4. Interface-specific
  5. Electrode-specific
Physics {
  # Global models
  Mobility( DopingDependence HighFieldSaturation )
  Recombination( SRH Auger )
}
Physics (Material="Silicon") {
  # Material-specific models
  Bandgap( Slotboom )
}

Mixed-Mode Simulations

Mixed-mode allows coupling rigorous TCAD device models with SPICE/PrimeSim HSPICE compact models in a circuit simulation.

System Section

The System section defines the circuit netlist, instantiated devices, and connections.

System {
  # Instantiate a TCAD device
  MOSFET nmos (gate=g drain=d source=s bulk=0)
  
  # SPICE elements
  Vsource_pset vdd (d 0) { dc = 5.0 }
  Vsource_pset vgg (g 0) { dc = 0.0 }
  Resistor_pset rload (d vdd) { resistance = 1e4 }
}

Device & Math Sections in Mixed-Mode

The Device section acts as a wrapper for the individual TCAD device definitions (containing its own File, Physics, and Electrode sections).

Performing Numeric Experiments

Boundary Conditions

Defined in the Electrode block. Supports Ohmic (default), Schottky (specify WorkFunction), floating, and resistive/distributed contacts.

Electrode {
  { Name="gate" Voltage=0.0 WorkFunction=4.6 }
  { Name="drain" Voltage=0.0 Resistor=100 }
}

Solve Commands

The Solve section orchestrates the numerical experiment (DC sweeps, transients, AC).

CommandDescription
Coupled { ... }Solves equations simultaneously (e.g., Poisson, Electron, Hole).
Quasistationary ( ... )Performs a DC bias sweep, ramping a voltage or parameter.
Transient ( ... )Performs a time-dependent simulation (e.g., pulse response).
ACCoupled ( ... )Performs small-signal AC analysis at a specific DC bias.
HBCoupled ( ... )Harmonic Balance for large-signal RF steady-state.
Save (FilePrefix="name")Saves the current state to a .tdr and .plt file.
Load (FilePrefix="name")Loads a previously saved state.

Example: DC Sweep

Solve {
  # Initial guess
  Coupled (Iterations=100) { Poisson }
  Coupled { Poisson Electron Hole }
  
  # Ramp Drain
  Quasistationary (
    Goal { Name="drain" Voltage=5.0 }
    InitialStep=0.1 MaxStep=0.5 MinStep=1e-4
  ) { Coupled { Poisson Electron Hole } }
}

Numeric & Software Issues

The Math section controls the numeric solvers. Use Extrapolate to speed up sweeps by guessing the next solution based on the previous two steps.

Math KeywordDescription
Method = Blocked / ILSSpecifies the linear solver (Blocked, ILS, PARDISO, Super).
Iterations = 20Maximum number of Newton iterations per solve step.
ExtrapolateEnables polynomial extrapolation for initial guesses during sweeps.
Number_of_ThreadsEnables SMP parallelization (e.g., Number_of_Threads = 8).
RelErrControlEnables relative error control (useful for low currents).
Transient = TRBDFSpecifies the transient integration method (TRBDF or BackwardEuler).

Physics in Sentaurus Device

Sentaurus Device solves the fundamental semiconductor equations: the Poisson equation and the continuity equations for electrons and holes.

  • Carrier Transport: Supports Drift-Diffusion (default), Thermodynamic (includes temperature gradients), and Hydrodynamic (carrier energy conservation).
  • Mobility Models:
    • DopingDependence (Masetti/Arora)
    • HighFieldSaturation (Canali)
    • Enormal (Lombardi) for surface scattering
  • Generation/Recombination: Includes SRH (doping and field-dependent), Auger, Avalanche (UniBo/VanOverstraeten), and Band-to-Band Tunneling (Hurkx/Dynamic/NonLocal).
  • Quantum Confinement: MLDA (Modified Local Density Approximation), eDensity/hDensity Schroedinger, Density Gradient.
  • Traps: Interface and bulk traps defined with discrete or continuous energy distributions (Gaussian, Exponential).

Numeric Methods & PMI

The Physical Model Interface (PMI) allows users to write custom C++ models (e.g., custom mobility or recombination models) and link them dynamically to Sentaurus Device at runtime without needing the source code of the simulator.

Equations are discretized using the Box Method on spatial grids, generating sparse linear systems solved by highly optimized direct (PARDISO, Super) or iterative (ILS) solvers.
